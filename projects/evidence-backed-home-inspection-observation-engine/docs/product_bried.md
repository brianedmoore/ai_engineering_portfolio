# Product Brief

## Project Name

Evidence-Backed Home Inspection Observation Engine

## Problem

Home inspectors document important findings in the field using photos, typed notes, and audio narration. These inputs often happen quickly and in different orders.

After the inspection, the inspector still has to turn those raw field inputs into clear, professional report language.

This creates repetitive work, slows down report delivery, and can lead to inconsistent wording across observations.

## Target User

The target user is a home inspector documenting an issue that needs to appear in a home inspection report.

## Core Product Idea

The system converts unordered field inputs into one structured inspection observation.

A complete observation requires:

- At least one photo as visual evidence
- At least one typed description or audio transcript
- A structured output ready for inspector review

## Version 1 Scope

Version 1 focuses on a single observation.

The system should accept:

- One or more photos
- Typed text, audio transcript, or both

The system should produce:

- Observation title
- Room or area classification
- Home system classification
- Component
- Defect type
- Severity
- Safety flag
- Professional report description
- Plain-English summary
- Recommended action
- Responsible professional
- Estimated cost range
- Human-review flag
- Missing-information list

## Product Principle

The AI does not replace the inspector.

The AI drafts, structures, and classifies the observation. The inspector reviews, edits, and approves the final report language.
